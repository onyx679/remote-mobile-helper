# 当前工作流程

## 目标

当前保留两条有效流程：

1. 从 `scrcpy` 手机画面识别单词并判断 YES / NO。
2. 手机拍照后自动传到电脑剪贴板，方便直接粘贴。

不使用浏览器 DOM。

当前真实链路是：

```text
手机摄像头拍到题目
-> 手机屏幕显示相机画面
-> scrcpy 把手机屏幕投到电脑
-> 从 scrcpy 窗口视频帧抓图
-> OCR 识别题目中的英文单词
-> 本地词库判断 YES / NO
```

DOM 辅助已删除，因为它只能读取电脑浏览器里的真实网页文字；现在要识别的是手机画面里的图片文字，DOM 对这个场景没有用。

## 主用程序

- `start-scrcpy-small.ps1`：打开 scrcpy 小窗口。
- `start-word-answer.cmd`：启动单词识别窗口。
- `stop-word-answer.cmd`：停止单词识别。
- `watch-phone-answer-fast.ps1`：识别启动配置。
- `watch_phone_answer_fast.py`：核心识别程序。
- `judge_english_word.py`：判断单词是否常见英文词。
- `det_word_bank.py`：单词修正规则和判定辅助。

## 当前识别配置

`watch-phone-answer-fast.ps1` 当前默认配置：

```powershell
$Serial = "adb-2252475e-baGT88._adb-tls-connect._tcp"
$Crop = "0.58,0.54,1.00,0.76"
$ScreenRegion = "scrcpy"
$PollSeconds = 0.15
```

关键点：

- `$ScreenRegion = "scrcpy"` 表示从 scrcpy 窗口视频帧抓图。
- 不使用 ADB `screencap`，因为它通常要几秒。
- 不使用浏览器 DOM。
- 当前目标是 2 秒内输出，实测通常约 0.3-0.5 秒。

## 常用启动顺序

1. 确认手机已通过 ADB 连上：

```powershell
.\check-devices.ps1
```

2. 打开 scrcpy：

```powershell
.\start-scrcpy-small.ps1
```

3. 启动识别：

```cmd
start-word-answer.cmd
```

4. 停止识别：

```cmd
stop-word-answer.cmd
```

## 输出文件

- `latest-result.json`：最近一次识别结果和耗时。
- `latest-answer.txt`：最近一次 YES / NO。
- `latest-crop.png`：实际送进 OCR 的识别区域。
- `latest-word-area.png`：识别程序保存的词区域。

## 手动调整识别框

如果识别位置不对：

```cmd
select-ocr-crop.cmd
```

查看当前识别框：

```cmd
show-ocr-crop.cmd
```

调整后需要把新的 crop 写回 `watch-phone-answer-fast.ps1` 的 `$Crop`。

## 拍照复制到剪贴板

这是另一条独立流程，不影响单词识别。

真实链路是：

```text
手机拍照
-> ADB 检测手机相册新照片
-> 拉取照片到电脑 camera-inbox
-> 写入 Windows 剪贴板
-> 在目标应用里 Ctrl+V 粘贴
```

- `start-photo-clipboard.cmd`：监听手机新照片并复制到剪贴板。
- `stop-photo-clipboard.cmd`：停止监听。
- `sync-camera-photos.ps1`：核心照片同步脚本。
- `copy-photo-to-clipboard.ps1`：手动复制已有照片。

启动：

```cmd
start-photo-clipboard.cmd
```

停止：

```cmd
stop-photo-clipboard.cmd
```

同步到本地的原图会保存在 `camera-inbox`。如果启用了旋转，旋转后的副本会保存在 `camera-rotated`。

## 出问题时先查

- `latest-result.json` 如果是 `no_frame`：先确认 scrcpy 窗口还开着，标题是 `Android-Remote-WiFi`。
- 如果识别慢：确认 `watch-phone-answer-fast.ps1` 里 `$ScreenRegion` 还是 `"scrcpy"`。
- 如果识别错位置：看 `latest-crop.png`，然后重新画框。
- 如果手机断开：重新连接 ADB，再重启 scrcpy 和识别窗口。
